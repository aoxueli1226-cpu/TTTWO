# TT - 优化目标文档

## 📱 项目定位

这是一个**真实可用的个人效率工具APP**，核心价值是：
- **桌面小卡片**：待办清单直接展示在手机桌面，无需打开APP
- **AI辅助创建**：通过拍照/截图/悬浮球快速创建待办事项
- **多端同步**：手机号登录，数据云端同步

> "我要做的是一个真正可以自己在自己手机上使用的一个真实的APP，不是做一个展示的东西。我是要做真正我可以用，然后我可以推广给别人让他们也可以用的APP。"


## 🔴 P0 - 必须实现（核心功能）

### 1. 桌面小卡片 (Widget) - **当前无法使用**

**现状问题**：
- ❌ 小卡片不存在或不可见：桌面上无法添加/看不到TT小卡片组件
- ❌ 无法交互：即使有卡片，点击也只能打开APP，不能直接在桌面上操作
- ❌ 无实时更新：完成事项后卡片内容不会自动刷新

**目标**：实现真正的Android桌面小卡片，让待办事项**直接悬浮在手机桌面背景上**，无需打开APP就能查看和操作

> "为什么叫他桌面小卡片，意思是他就是展示在APP外的，类似那种主题小卡片组件一样的，直接是悬浮在手机桌面背景下的，所以才叫桌面小卡片。这是我最初想要实现的功能，因为我们现在想要看到一些待办清单，一些待办APP都要点进去才能看到一些事项，这很不方便，所以我就想让他直接展示在桌面上。"

#### 1.1 基础卡片功能
- [ ] 在手机桌面添加TT小卡片组件（长按桌面 → 添加小部件 → 找到TT）
- [ ] 卡片直接展示待办清单，不是"点击打开APP才能看到内容"
- [ ] 支持不同尺寸（2x2显示最近3条、4x2显示最近5条、4x4显示完整清单）
- [ ] 卡片样式可配置（主题色、字体大小、是否显示已完成）

#### 1.2 卡片交互
- [ ] **直接在卡片上点击复选框标记完成**，无需打开APP
- [ ] 点击卡片空白区域打开APP查看详情
- [ ] 完成事项后卡片实时刷新（1秒内更新）
- [ ] 支持滑动查看更多（大尺寸卡片）

#### 1.3 数据同步
- [ ] 卡片数据与APP内数据实时同步
- [ ] 支持多卡片同时存在（工作卡片、生活卡片）
- [ ] 可配置每张卡片显示不同分类的待办

#### 1.4 卡片显示规则（时间段任务标注）

**核心原则**：桌面小卡片只显示**当天的任务**，但通过不同标注区分任务类型：

> "桌面小卡片上因为它只显示本天的要完成的任务，如果是一个时间段内的任务就在这个任务后面标注多天，不然的话就是可以在后面标注一个具体到几小时几分的这个时间点。"

| 任务类型 | 显示示例 | 说明 |
|---------|---------|------|
| **全天任务**（无具体时间） | `开会` | 只显示标题，无时间标注 |
| **多天任务**（跨天） | `出差北京 ⏱️多天` | 标题后加"⏱️多天"标注 |
| **具体时间点任务** | `见客户 14:30` | 标题后显示具体时间（时:分） |
| **重复任务**（每日/每周） | `晨会 🔁` | 标题后加循环符号 |

**卡片上显示示例：**
```
📅 3月19日 周三

☐ 准备演示文稿 14:00
☐ 项目评审会
☐ 客户沟通 16:30
☐ 出差北京 ⏱️多天
☐ 团队周报 🔁

-------------------
+ 添加新任务
```

### 2. 悬浮球功能（快捷创建）

**现状**：无

**目标**：系统级悬浮球，随时调起创建事项界面

> "为什么要这个悬浮球？就是因为我希望我能在打开某些页面的时候，他可以一键扫描当前页面，然后总结出需要的内容，一键创办这个新的事项。或者说我们看到这个页面突然想起来要创办一个新的事项，然后我们打开这个悬浮球，对照这个页面就可以输入我们想要的数据，而不是再打开一部APP，这样就会有折损。"

> "这个悬浮球我希望他的打开按钮是在这个手机下拉框这个组件。比如我这个是小米15，我手机下拉框之后就有这个组件，组件里就有飞行器呀、蓝牙、手电筒等等这些小组件，我希望就是放在这里。其他安卓手机也是类似的。"

**说明**：此功能实现较复杂（需要系统权限），如果当前版本难以实现，放入后续迭代


### 3. AI辅助创建事项 - **当前完全不可用，需彻底重构**

**现状问题**：
- ❌ API调用是假的：用户配置了真实的API密钥（如DeepSeek、阿里云）但根本没生效
- ❌ 逻辑错误：用户输入什么，系统就直接把文本复制到"注意事项"字段，根本没有AI处理
- ❌ 没有真实调用：代码里可能只是保存了配置，没有实际发送HTTP请求到AI服务商
- ❌ 无法一键创建：不能根据用户意图自动提取标题、时间、分类等信息

> "这个AI啊，其实我添加了我真实的比如说这个DeepSeek的API密钥，它其实还是不能用。我现在那个详情页里面的AI窗口，只是我说什么它就会自动的把这个我填输进去的内容给放到注意事项里，这不是我要的呀，这根本没有办法AI一键创建一键修改。"

**根本原因**：
- API请求没有真正发出，或地址配错（如阿里云灵积地址和DeepSeek地址混用）
- 返回结果没有被正确解析和结构化
- 缺乏错误提示，用户以为配置了就能用，其实一直是假的

**目标**：实现真实的AI调用，让AI真正理解用户输入并生成结构化待办

#### 3.2 AI理解与结构化输出

### 4. 未完成事项分级展示（按真实日期分组）

**现状问题**：
- ❌ 所有事项平铺展示，没有层级关系
- ❌ 无法按时间维度筛选（上周/本周/今日/逾期）
- ❌ 大量事项时难以快速找到重点

> "未完成这里，我要的是像标签页那样有一级二级三级。第一级就是上一周还是本周，第二级就是按天分，第三级就是具体的没完成的事项都放到。"

**目标**：实现**三级标签页结构**，按真实日期（月/日）分组展示待办清单

#### 4.1 三级结构设计

**第一级：时间维度（顶部标签页）**
- 📅 **上一周**（显示上周的未完成事项，即逾期事项）
- 📆 **本周**（显示本周的所有未完成事项）
- ⏰ **今日**（聚焦今天要完成的事项）
- 🗓️ **自定义**（选择任意日期范围）

**第二级：按具体日期分组（列表头部）**
```
本周视图（示例）：
├── 3月15日 (3项未完成)
│  ├ 项目会议
│  ├ 提交报告
│  └ 代码审查
├── 3月17日 (2项未完成)
│  ├ 晨会
│  └ 客户沟通
└── 3月19日 (1项未完成)
  └ 准备演示文稿

上一周视图（示例）：
├── 3月8日 (2项逾期)
│  ├ 完成方案设计
│  └ 团队周报
├── 3月10日 (1项逾期)
│  └ 采购申请
└── 3月12日 (3项逾期)
  ├ 测试用例
  ├ 代码review
  └ 文档更新
```

**重要说明**：
- ✅ 日期格式：使用**月和日**（如"3月15日"），**不是**星期几（周一/周二）
- ✅ 上一周：显示的是**具体的日期**，按日期倒序排列
- ✅ 本周：显示的是**具体的日期**，按日期顺序排列
- ✅ 逾期事项：在上一周视图中红色高亮显示

**第三级：具体事项（可折叠/展开）**
- 每个事项显示：**复选框 + 事项标题**（**不显示时间**，只显示名称）
- 事项排序：默认按**截止时间先后**排序（但界面上只显示标题）
- 长按事项显示操作菜单（编辑、删除、推迟）
- 左滑标记完成，右滑推迟到明天

#### 4.2 交互要求
- [ ] 默认展开今日事项，其他日期折叠
- [ ] 点击日期头可展开/折叠该日期下的事项
- [ ] 支持批量操作（一键完成当日所有事项）
- [ ] 拖拽调整事项顺序（仅影响显示顺序，不改变截止时间）
- [ ] 下拉刷新同步最新数据
